# GFTI

GFTI 是一个《少女前线》同人向人格匹配站点。玩家可以在站内查看角色类型资料，也可以完成一份 28 题测试，得到自己的四码类型、七维分数，以及最接近的人形匹配结果。

它的用途不是做“官方设定考据”，也不是做严肃心理测量，而是把《少女前线》角色气质、队伍关系、行动风格和玩家自我投射放到同一个轻量玩法里，方便讨论、创作和分享。

## 玩家能在这里做什么

- 浏览热门人形的 `GFTI` 类型卡、七维画像和判定依据。
- 完成测试，获得自己的四码结果和 1 组主匹配角色。
- 查看前 4 名相近人形，方便做扩写、代入或同人创作参考。
- 导出本地 `JSON` 结果，之后重新导入恢复测试结果页。
- 在结果页点击“查看官方 Wiki”，跳转到对应官方或社区资料站入口。（暂时维护中，后续完善人形列表后统一补充链接）

## 技术实现

项目采用 `HTML + CSS + Vanilla JavaScript` 实现，没有引入前端框架，也没有后端依赖。所有页面内容都由本地数据文件驱动，浏览器打开后直接完成渲染、测试计算和结果展示。

首页通过 `app.js` 读取角色数据，按队伍和关键词筛选热门卡片，并渲染来源标签与版权说明。测试页通过 `test-app.js` 维护当前题号、答案数组和结果缓存，在前端完成整个问答流程。

## 测试原理

测试结果由三层逻辑组成：

1. 每个用户初始分数都是同一组七维基线值：`50`。
2. 每道题的每个选项都对应若干维度增减，答题后按权重累计，最终分数限制在 `0-100`。
3. 系统把你的七维分数与角色库的七维画像做距离比较，取最近的角色作为主匹配，并生成前 4 名相近结果。

当前实现里，题目权重保存在 `data/test-data.js`，计算函数位于 `test-app.js`：

- `calcScore()`：把题目选项转换成七维分数。
- `scoreToCode()`：按阈值规则把七维结果映射成四码类型。
- `distance()`：计算玩家分数与角色画像之间的距离。
- `rankMatches()`：按距离排序，生成主匹配和相近匹配。

## 主要指标

GFTI 当前使用 7 个核心维度：

- `command`：指挥。更偏主动接管、发号施令还是让渡主导权。
- `reserve`：保留。边界感、警惕性、信息暴露意愿的高低。
- `discipline`：纪律。对秩序、规则、执行标准的重视程度。
- `adaptation`：适应。临场变通、重写路径、调整策略的能力。
- `warmth`：温度。对外显亲和、安抚、共情和情感表达倾向。
- `teamTrust`：队信。对“自己人”的投入、托底和协作信任倾向。
- `volatility`：波动。情绪热度、行动爆发性和状态起伏幅度。

四码 `GFTI` 不是直接抄写 MBTI，而是根据这 7 个维度的阈值组合生成，用来快速概括角色或玩家在“行动方式 + 情感风格 + 关系姿态”上的整体倾向。
